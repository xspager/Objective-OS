I believe that the design of the wonderful Linux, if it exist :), is poor. But many times there things that can't be included on Linux without change many things. And not just Linus do this, far from this, many Open Source projects

What is it?

This are my ideas about how an operating system supposed to be. An Operating System written mostly in Objective-C programing language, but in C too, and a little in assembly

Why write a new operating system, an write it in Objective-C?

First, 'cause I like operating systems and I want to know if I agree with Linus about write am OS. Second because it will make me go really deep on the computers, compute programing projects and in the software community. But there other reasons, like:

  • I believe in the end of the Linux. As we say here on Brazil, "o Linux j√° deu o que tinha que dar!" (the Linux already give what it supposed to give), and I believe that the community could make more than what Linux now could be.
  • The Objective-C language give to the programmer the power and freedom of C (a hi-low level language) with the flexibility of the object oriented programing in a way that only Objective-C could do.
  • GCC compiles Objective-C, and it is available to any main computer architecture.

Features wanted

The order don't reflect the priority of the features

  1. Support to the Linux drivers to avoid problem with no compatible hardware
  2. An Framework to write a kernel written in Objective-C
  3. Kernel architecture independent OS. (no meter if is micro, macro, monolithic, exo,...) :)))))
  4. Internalization from the start. (Unicode, UTF-8)
  5. Implementation of the OpenStep. (from GNUstep?)
  6. NO X!!! To compatibility could be written a library to emulate X
  7. Very hi integration between application
  8. Extract all power from the resources
  9. Distributed system. (in a Plan 9 way?)
  10. Use of Plan 9 C compiler like syntax. It create the need of a task.
  11. The system should run like it wasn't software, explaining, the OS shouldn't crash, freeze, reboot, it should behave like a hardware, question - answer, input - output.
  12. Use GRUB as boot loader
  13. Persistent enviroment storage,
  14. Every thing could be changed by user, in two levels, a common user level, and a power user level

Support to the Linux drivers

Could be implemented using glue code like in OSKit, after the system became useful, the drive code could be ported, rewritten or written.

Framework to write kernels, write in Objective-C

Would be the base to the OS kernel, and the future drivers

Internalization

Implementation of Unicode as the system character encoding
